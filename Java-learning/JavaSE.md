## 🔧 方法（Method）

### `public static int sum()` 与 C++ 函数的对比

Java 中的 `public static int sum()` 类似于 C++ 中的全局函数 `int sum(){}`，但 Java 更强调访问控制和静态特性。

### `public` 的含义

表示方法可以被任何类访问（公共权限）。

### `static` 的含义

表示方法属于类本身，而非实例，可通过类名直接调用，无需创建对象。

### 关键区别

- **static** 解决「是否需要创建对象」的问题
- **访问修饰符**（如 public）决定「能否被其他类访问」

### 💡 通俗理解

- **static方法**像「公共工具」，直接可用
- **普通方法**像「个人物品」，需先有对象才能使用

### 📌 知识点补充

1. **方法重载（Overload）**
    - 同一个类中，方法名可以相同。
    - 只要**形参列表不同**（参数个数、类型、顺序不同），编译器就能区分。
2. **基本数据类型的类型转换**
    - 小范围 → 大范围：一般**自动类型提升**（隐式转换）。
    - 大范围 → 小范围：需要**强制类型转换**（显式转换），并注意可能发生**数据溢出**。
    - 示例：
        
        ```java
        int b;
        byte a = (byte) b;
        ```
        
3. **表达式中的类型提升**
    - 在算术表达式中，`byte`、`short`、`char` 等小范围类型参与运算时，会先**自动提升为 int**（再进行计算）。

### ✅ 今日总结

- 理解了 `public` 和 `static` 的作用，能判断一个方法是否需要创建对象调用。
- 了解了方法重载的判断依据：看形参列表是否不同。
- 梳理了基本数据类型的自动转换与强转规则，知道强转可能带来溢出问题。
- 记住了表达式里小范围类型会先提升，避免因为类型不匹配导致的编译错误。

## 🎯 今日学习内容

## ⌨️ Scanner 键盘输入（3步）

1. **导包：**
    
    ```java
    import java.util.Scanner;
    ```
    
2. **创建对象：**

## 📌 注意事项

- `Scanner` 的类名首字母要大写，导包也要写 `java.util.Scanner`。
- 用完后可以关闭资源：
    
    ```java
    sc.close();
    ```
    

## ➕ 运算符补充：`+` 的两种作用

- `+` 既可以做**算术运算符**，也可以做**字符串连接符**。
- 规则：**能做数值运算就先做运算**，否则做连接。
    
    ```java
    System.out.println(1 + 2);        // 3
    System.out.println(1 + "2");      // "12"（连接）
    System.out.println("1" + 2 + 3);  // "123"（从左到右依次连接）
    System.out.println(1 + 2 + "3");  // "33"（先算 1+2，再连接）
    ```
    
- **数字 + 单引号字符**：会按字符的 **ASCII/Unicode 编码值**参与运算。
    
    ```java
    System.out.println(1 + 'a'); // 98（'a' 的编码是 97）
    ```
    
- **数字 + 双引号字符串**：会进行字符串连接。
    
    ```java
    System.out.println(1 + "a"); // "1a"
    ```
    

## ✅ 今日总结

- 会使用 `Scanner` 从键盘读取 `int` 和 `String` 类型数据。
- 学习了一些运算符。

# 📝 Java学习笔记 - 第4天

## 🎯 今日学习内容

今天学习了**逻辑运算符**，并对比了 Java 与 C/C++ 的写法差异。

## 🧠 逻辑运算符：`&` / `&&`，`|` / `||`

在 Java 里，`&` 和 `|` **不只是**位运算符，在操作数为 `boolean` 时也可以做逻辑运算；但它们与 `&&`、`||` 的关键区别是：

- `&&`、`||`：**短路运算**（左边就能决定结果时，右边不再计算）
- `&`、`|`：**不短路**（两边都会计算）

### ✅ 与 C/C++ 的对比要点

- Java 中：
    - 逻辑与：既有 `&&` 也有 `&`
    - 逻辑或：既有 `||` 也有 `|`
- 常规写逻辑判断时，**更推荐用 `&&` 和 `||`**（更安全也更高效）。

### 🔍 示例：短路 vs 不短路

```java
int a = 0;

// 短路与：a == 0 为 true，继续判断右边；但如果左边为 false，右边不会执行
if (a != 0 && (10 / a) > 1) {
    System.out.println("OK");
}

// 不短路与：无论左边真假，右边都会执行（可能导致异常）
// if (a != 0 & (10 / a) > 1) { }
```

### 📌 什么时候用 `&` / `|`（布尔场景）

当你**明确希望右边表达式一定执行**（比如右边有必须执行的逻辑）时，才考虑 `&` 或 `|`。

## 🔀 Java vs C/C++：`if` 与 `switch` 的区别整理

### 1) `if` 的关键区别

- **基本格式**：
    - C/C++：`if (condition) { ... }`
    - Java：`if (condition) { ... }`
    - 结论：**语法形式基本一致**。
- **条件表达式类型**：
    - C/C++：`condition` 可以是整数、指针等（**非 0 即真**）。
    - Java：`condition` **必须是 `boolean`**（不允许用 `int`/指针当条件）。
    - 例子：
        
        ```cpp
        // C/C++ 可以
        if (1) { }
        if (ptr) { }
        ```
        
        ```java
        // Java 不允许
        // if (1) { }            // 编译错误
        // if (someInt) { }      // 编译错误
        ```
        
- **悬挂 else（dangling else）规则**：
    - C/C++ 和 Java 都是：`else` 匹配**最近的、尚未匹配的 `if`**（规则一致）。

### 2) `switch`：现代 Java（17/21+）的几个重点变化

> 备注：下面是“现代 Java”语法概览，是否能用取决于你的 JDK 版本与编译选项。
> 

#### 2.1 `switch` 从“语句”变成“表达式”（Switch Expressions）

- 现代 Java 可以直接用 `switch` **返回值**：
    
    ```java
    String feel = switch (day) {
    	case MONDAY, FRIDAY -> "累";
    	case SATURDAY, SUNDAY -> "爽";
    	default -> "麻了";
    };
    ```
    
- `->` 形式**自带 break 效果**，不会穿透（fall-through）。

#### 2.2 模式匹配（Pattern Matching，类型匹配）

- Java 的 `switch` 可以匹配类型，并“顺便完成” `instanceof + 强转`：
    
    ```java
    Object obj = "Hello World";
    
    switch (obj) {
    	case Integer i -> System.out.println("这是一个整数: " + i);
    	case String s && s.length() > 5 -> System.out.println("长字符串: " + s.toUpperCase());
    	case String s -> System.out.println("短字符串: " + s);
    	default -> System.out.println("未知类型");
    }
    ```
    
- 其中 `&&` 这种额外判断可以理解成“带条件的匹配”（guard）。

#### 2.3 穷举性检查（配合 sealed classes 更安全）

- 如果对一个 sealed 层级做 `switch`，Java 编译器可以强制你把所有分支写全。
- C/C++ 一般更依赖 `default` 或者 warning。

### 3) 对比小结（速记）

- `if`：Java **必须 boolean**；C/C++ **非零即真**。
- `switch`：
    - C/C++：默认可能穿透，需要 `break`。
    - Java：`->` 不穿透；也可以用 `:` 写出穿透逻辑。
    - Java（新）：可当表达式返回值；可做类型/模式匹配（更强）。
    
    ```java
    int day = 1;
    int mood = switch (day) {
        case 1          -> 100;
        case 2, 3, 4    -> 50;  // 多个条件合并
        default         -> 0;
    }; // 注意这里有分号，因为它是一个赋值语句
    ```
    

## ✅ 今日总结

- 理解了 Java 中 `&&`、`||` 是短路逻辑运算。
- 明白了 `&`、`|` 在布尔运算时也能用，但**不会短路**，需要注意右侧表达式可能带来的风险。
- 梳理了 Java 与 C/C++ 在 `if` 条件类型、以及现代 `switch` 写法上的差异。
